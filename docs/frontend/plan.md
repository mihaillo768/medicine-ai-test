# План разработки фронтенда (Vue)

Документ задаёт **последовательность работ** по вёрстке и сборке интерфейса медицинского центра. Источник требований к страницам, блокам и критериям результата — **[ТЗ в Google Docs](https://docs.google.com/document/d/1--N5B8rCHCvRSyfK8uuXMWmFgGNsZGkyetUHX1KOeHc/edit?tab=t.0)**. Визуальная и структурная опора — макеты в **[Figma](https://www.figma.com/design/SIOFqk6OdgyZCgq7Hgovd5/Untitled?t=KH7MP7x11F1j3yvy-0)** (страница «Макеты»). Дополнительный контекст: [docs/design/ia-map.md](../design/ia-map.md), [AGENTS.md](../../AGENTS.md).

**Исходники приложения** — в каталоге **`frontend/`** в **корне репозитория** (этот файл — только план работ, он лежит в `docs/frontend/`). В `frontend/` — только клиентское приложение на **Vue** (сборка, маршрутизация, компоненты, стили, статика). Backend, CMS, API и интеграции туда **не входят**; формы и кнопки записи на этапе прототипа — с **локальной валидацией** и имитацией отправки (без реального сервера), если иное не появится в ТЗ.

---

## Принципы выполнения

1. **Сверка с ТЗ:** перед финализацией текстов, состава меню и обязательных блоков сверяйся с Google Doc; при расхождении макет/код — уточняй по документу.
2. **Пара Desktop + Mobile:** для каждой из 11 страниц есть отдельные фреймы; в коде — одна разметка с **адаптивом** (контрольные точки по макетам: desktop **1440** с контентом **1280** и полями **80**; mobile **390** с контентом **358** и боковыми **16**, шаг сетки **8** px — как в шаблонах Figma).
3. **Картинки:** в макетах нет финальных фото/иллюстраций. При вёрстке **подбирать изображения из открытых источников** (стоки с допустимой лицензией: Unsplash, Pexels и т.п.), тематика — взрослая медицина, спокойная эстетика; **альтернатива** — переданные вручную образцы от заказчика. Приоритет — **самостоятельный подбор**, согласование с ТЗ по смыслу блоков.
4. **Логотип:** в макетах вместо бренда — зелёный квадрат. В реализации — **оригинальный знак и/или логотип в SVG**, придуманный и сверстанный в рамках проекта (единый файл или компонент), в духе спокойного медицинского центра, без копирования чужих товарных знаков.
5. **Единый визуальный язык:** компоненты и отступы согласованы между страницами (как в Figma: кнопки, поля, карточки услуг и врачей, хедер/футер).
6. **Демонстрация после этапа:** по завершении **каждого** этапа плана нужно **показывать результат** — в том числе запускать приложение (`npm run dev` в `frontend/`) и **открывать проект в браузере**, чтобы визуально подтвердить работу интерфейса и маршрутов (при необходимости — на ширинах desktop и mobile).

## Этап 0. Подготовка репозитория (первый исполнительный шаг после утверждения плана)

- [x] Инициализировать в **`frontend/`** (корень репозитория) проект **Vue 3** (рекомендуемо **Vite** + **Vue Router**; при необходимости общего состояния — **Pinia**, минимально).
- [x] Настроить единый стиль кода (ESLint / Prettier по договорённости команды).
- [x] Структура папок внутри `frontend/` (ориентир): `components/` (ui, layout, sections), `views/` или `pages/`, `assets/` (в т.ч. `logo.svg`), `styles/` (токены, глобальные стили), `router/`.
- [x] Подключить шрифт **Inter** (как в макетах) — через `@font-face` или провайдер; fallback для кириллицы.

- [x] Задать глобальные стили: `Heading/H1–H3`, `Body/Regular`, `Caption/Regular`, `Button/Label` — соответствие размерам и весам из макетов.

- [x] Кнопки: **Primary**, **Secondary**, **Ghost** (состояния default / hover / focus / disabled — ориентир: фрейм `_Ref / Component states` на «DS / Компоненты»).
