# Синхронизация макета Figma с фронтендом

Чеклист правок в [рабочем файле Figma](https://www.figma.com/design/SIOFqk6OdgyZCgq7Hgovd5/Untitled?t=KH7MP7x11F1j3yvy-0), чтобы макеты соответствовали **фактическому виду** приложения в каталоге [`frontend/`](../../frontend/) (Vue 3). Источник формулировок и структуры блоков на этом этапе — **код**; [ТЗ в Google Docs](https://docs.google.com/document/d/1--N5B8rCHCvRSyfK8uuXMWmFgGNsZGkyetUHX1KOeHc/edit?tab=t.0) остаётся источником по продукту — при расхождениях с ТЗ зафиксировать в задаче, не подменять молча.

**Как сверять:** инвентаризация Figma (ниже) + браузер на ширинах **~390** и **~1440** px (`npm run dev` в `frontend/`) + при необходимости данные в [`frontend/src/data/`](../../frontend/src/data/).

## 4. Главная — фреймы `Главная / Desktop`, `Главная / Mobile`

В [docs/design/plan.md](plan.md) (п.5) описан порядок Hero → услуги → врачи → запись → trust (три текстовые колонки). **На сайте другой порядок и смысл блоков** — ориентир [`frontend/src/views/HomeView.vue`](../../frontend/src/views/HomeView.vue).

**Целевой порядок секций**

1. **Hero** ([`PageHero.vue`](../../frontend/src/components/sections/PageHero.vue)): eyebrow «Добро пожаловать»; заголовок «Медицинский центр для взрослых пациентов»; подзаголовок про консультации и шаги; изображение из [`imageUrls.ts`](../../frontend/src/data/imageUrls.ts); кнопки **«Записаться»** (primary), **«Услуги»** (secondary).
2. **Полоса доверия:** три метрики — **15+** («лет амбулаторной практики»), **40+** («специалистов очного приёма»), **8:00–21:00** («ежедневная запись без выходных»). Не три абзаца «колонки доверия» без цифр.
3. **SectionIntro:** заголовок «Комплексный подход», абзац про терапевтическую оценку и специалистов.
4. **Популярные услуги:** заголовок секции, описание из `SectionHeading`, три карточки; на карточке CTA **«Подробнее»** (secondary); внизу **«Все услуги»** (ghost).
5. **Врачи:** заголовок «Врачи», описание из кода; на карточках превью **«Записаться»** (primary); **«Все врачи»** (ghost).
6. **Форма записи** с заголовком «Запись на приём» и лидом про контакты и раздел контактов (`anchor` `home-booking`).
7. **BottomCta:** «Нужна консультация?», описание, кнопки «Записаться» и «Контакты».

**Чеклист в Figma**

- [ ] Переставить/перерисовать секции в `_Main / Content` обоих фреймов под порядок выше.
- [ ] Заменить блок trust на числовую полосу с подписями как на сайте.
- [ ] Выравнять все подписи кнопок и тексты секций с `HomeView.vue`.
