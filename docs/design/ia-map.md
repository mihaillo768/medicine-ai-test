# Карта экранов (информационная архитектура)

Источник названий и состава: [ТЗ (Google Doc)](https://docs.google.com/document/d/1--N5B8rCHCvRSyfK8uuXMWmFgGNsZGkyetUHX1KOeHc/edit?tab=t.0), раздел «Минимальный состав страниц» и «Допускается часть внутренних страниц…».

В макетах проекта ([Figma](https://www.figma.com/design/SIOFqk6OdgyZCgq7Hgovd5/Untitled?t=KH7MP7x11F1j3yvy-0)) эта карта должна быть продублирована на отдельной странице **IA / Оглавление** во фрейме **`_IA — карта экранов`**. Создание во фрейме: вручную по таблице ниже, через **figma-mcp-go**, либо скриптом [`figma-create-ia-map.js`](figma-create-ia-map.js) при наличии MCP с **Plugin API** (`use_figma` + skill **figma-use**).

**Связь с п.1 плана:** каждый из 11 экранов далее верстается как пара **Desktop + Mobile** на базе шаблонов страницы «Макеты»: `_Template / Desktop 1440` и `_Template / Mobile 390`.

## 11 экранов и уровень проработки

| № | Экран (как в ТЗ) | Уровень |
|---|------------------|---------|
| 1 | Главная | Детально по ТЗ |
| 2 | Услуги | Детально по ТЗ |
| 3 | Страница отдельной услуги | Детально по ТЗ |
| 4 | Врачи | Детально по ТЗ |
| 5 | Страница врача | Детально по ТЗ |
| 6 | О центре | Типовой шаблон |
| 7 | Цены | Типовой шаблон |
| 8 | Контакты | Детально по ТЗ |
| 9 | Блог / статьи (листинг) | Типовой шаблон |
| 10 | Страница статьи | Детально по ТЗ |
| 11 | 404 | Типовой шаблон |

«Типовой шаблон» означает достаточную проработку для согласованного продукта, без требования максимальной детализации, перечисленной в ТЗ для ключевых разделов.
